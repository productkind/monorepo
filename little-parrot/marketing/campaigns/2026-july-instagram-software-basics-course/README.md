# Instagram: Basics of Software for Vibe Coding (6 carousels)

Six Instagram carousels built from the **Basics of Software for Vibe Coding**
course (`vibe-coding-tech-00`) and its toolkits, shaped by
`productkind/marketing/strategy/audience-research.md` and the content strategy.

## Why these six (research trace)

- **"To be taken seriously in technical rooms" is a core want.** Our own
  learner said it best: "Key technical terms that will allow me to have
  confident conversations with the developers" (Corina). Every post here is
  vocabulary taught as a real term with a plain definition, per the
  teach-real-terms principle.
- **Precise words are credit savings.** "Add a card component" gets the
  thing she pictured; "make it look nicer" starts a guessing loop. The
  vocabulary posts carry the same ROI thread as the other campaigns.
- **Analogies lower the entry bar** (the bakery, the spreadsheet, the three
  languages), which is confidence-by-design: recognise, don't memorise.
- **All six repackage existing toolkit items or course mental models**
  (vocabulary cheat sheet, component prompt library, where-is-the-problem
  guide), per the reuse-first rule. Cheat sheets are the top saveable
  format.

## Posting notes

- Account: Little Parrot Instagram. Cadence 1 to 2 per week, so this set
  covers three to four weeks. Suggested order: post in numbered order; it
  follows the course's outside-in arc (whole app → what you see → how
  screens work → behind the scenes → the memory), with the troubleshooting
  map (3) placed after the bakery post it builds on.
- CTAs name the full course title (Basics of Software for Vibe Coding)
  and LittleParrot.app rather than pointing at the bio, because the bio
  holds only 5 links and we have more courses than slots. The UTM URL
  below is for wherever the course link is actually placed (a bio slot,
  stories, comment replies):

  (add `&utm_content=post-1` etc. per post for per-post tracking).
- The course's first challenge (Building Blocks of an Application) is free;
  CTAs say so.
- Carousel slides: 1080 x 1350 (4:5 portrait), same visual system as the
  other July sets.
- Overlap notes: post 3 (where is the problem) locates bugs by app layer;
  the debugging campaign's posts investigate clues with DevTools. They
