# Post 03 — Mon 8 Jun

**Angle:** The fix wasn't a better prompt.
**Transferable tip:** Build a staged pipeline with a checkpoint between each step.

---

When I first asked AI to write a course, I asked for the whole thing in one go. The result was confident, fluent, and not good enough to publish.

So I broke it into stages. First it turns my messy notes into an outline, and only the outline. I read it, fix what is off, approve it. Then it turns the approved outline into lessons. Each stage has to pass a short checklist before moving on.

The work got better, and oddly faster, because mistakes get caught while they are small. Fixing five bullet points is easier than finding the problem buried in three thousand words.

We have built about a dozen courses this way now. The quality holds because the process does, not because I got lucky on a prompt.

What surprises me is that I still reach for this even as the models get better. When the output is almost-but-not-quite, smaller steps with a checkpoint between them is still the thing that fixes it.
