# How to Save Lovable Credits: A Beginner's Guide to GitHub

This micro-course aims to give skills to save credits for vibe coders, who use tools like Lovable. It details how to use GitHub to track the code of their application and make edits without spending credits on Lovable or similar tools. This course also prepares the learner to use other AI agents to build software later on, such as Claude Code.

Underneath all of that is the benefit of demystifying software development. Seeing the file structure, watching commits accumulate, noticing what changed and when, all of it turns the black box into technical literacy.

## Target Audience

Non-technical women who build digital products using an AI app builder, like Lovable.

## Content

In this course only use the GitHub web UI in the browser, not the CLI, to avoid overwhelming the learners with too many tools at once. The main purpose is to show how to use less Lovable credits by making edits to the code on GitHub instead of in Lovable.

- How to save Lovable credits: editing website copy manually, instead of asking Lovable, using an AI agent outside of Lovable to fix bugs (in case Lovable is looping on the problem), understand how software is structured, so you can guide the AI better, make small UI changes.
  - Caveat: you can edit copy with "Visual edits" in Lovable, but it sometimes has performance issues (doesn't always function efficiently).
- What GitHub is and why it is useful for credit saving.
- How to create a GitHub account and set up the necessary things.
- How to connect your GitHub account to your Lovable project.
- Explain the structure of a GitHub repository.
- How to find the part of the code you want to edit using GitHub's search and file navigation features.
- Show how you can find a certain part of the code, by inspecting your app in the browser and looking at the Elements tab to find the path to the file the code is available in (data-lov-id), and the line number.
- Show the online code editor in GitHub and how to use it to make simple edits to the code.
- Explain how to edit a file on GitHub, then the process of adding that change to the repository.
- Explain the process of adding new code to the repository and how it will be available everywhere (e.g., in Lovable)
- Explain what the difference is between the code in GitHub and the code in Lovable, and how to keep them in sync (it takes a minute to see your external code change in Lovable).
- Explain what a commmit is.
- Explain what is the difference between GitHub and Git.

### Practical Example
Use Dalmie's example throughout the course. She is building her Book Club Organiser. Every time you explain how to do something on the UI of GitHub, set it up with a practical example what Dalmie wants to achieve, then explain how she can do that to achieve her goal.
