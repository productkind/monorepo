# How to Save Credits for Vibe Coders: GitHub Practices

This micro-course aims to give skills to save credits for vibe coders, using tools like Lovable. It details how to use GitHub to track the code of their application and make edits without spending credits on Lovable or similar tools.

## Target Audience

Non-technical women who built a digital product using an AI app builder like Lovable.

## Content

Start with GitHub web UI on the browser only, to avoid overwhelming the learners with too many
tools at once. With the main porpose to show how to use less credits by making edits to the code on
GitHub instead of Lovable.

- What is GitHub and why is it useful for tracking code changes.
- How to connect your GitHub repository to your project on Lovable.
- How to make edits to your code using GitHub and push changes to your project.
- Show the online code editor in GitHub and how to use it to make simple edits to the code.
- How to find the part of the code you want to edit using GitHub's search and file navigation
  features.
- Explain what is the difference between the code in GitHub and the code in Lovable, and how to keep
  them in sync.
- Explain what is the difference between Github and Git
- Explain what is a commmit

Show the command line and tell the masic flow of git commands, but do not go into details of how to
use the command line, as it may be overwhelming for non-technical learners. The goal is to give them
a basic understanding of how Git and CLI works to help them to learn to use tools like Claude Code.

- Explain how to install Git
- Explain what is CLI (bash) and explain if they need to install it for windows
- Explain the notion of local and remote repositories
- Explain the basic commands in bash for navigating directories (cd, ls, pwd) and how to use them to
  navigate the folders.
- Explain that you can run programs and scripts from the command line, and how to do it. With an
  example by running programs that they already know like a browser.
- Show what is a command line argument and how to use it with an example that they can relate to,
  like opening a specific website with a browser from the command line.
- Explain the basic Git workflow: git add, git commit, git push, git pull
- Explain how to install VSCode and how to use it to edit code locally
- Explain how to find and edit files using VSCode's file explorer and search features.
- Explain how to use Git integration in VSCode to manage your code changes and sync with GitHub.
