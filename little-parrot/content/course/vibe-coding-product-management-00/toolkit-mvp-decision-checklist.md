## MVP Decision Checklist

Use this checklist to validate your MVP scope before you start building. It helps you make sure you're building enough to be useful, but not so much that you lose focus.

### Does your MVP solve the core problem?

- [ ] **Can you trace every feature back to your problem statement?** If a feature doesn't help solve the problem you defined in challenge 1, it's not MVP. Save it for later.
- [ ] **Does it address your user's biggest frustration?** Look at your persona's frustrations. The MVP should directly tackle at least the top one.
- [ ] **Would your persona find this version useful, even without the extra features?** If Sarah could use the Book Club Organiser to collect suggestions and vote, that's useful. She doesn't need notifications, profiles, or reading progress tracking yet.

### Is the scope small enough?

- [ ] **Can you describe your MVP in one sentence?** If you need a paragraph, it's too big. For example: "A web app where book club members suggest books and vote on which to read next."
- [ ] **Does your MVP cover only 1-3 activities from your story map?** Most MVPs need just the core workflow, not every activity you mapped out.
- [ ] **Have you cut the nice-to-haves?** Go through your details one by one and ask: "Would the core experience still work without this?" If yes, cut it. You can always add it later.
- [ ] **Could you build and test this within a week?** If the scope feels like it would take much longer, you're probably including too much.

### Is it a complete experience?

- [ ] **Can your persona complete the full journey from start to finish?** An MVP shouldn't be a dead end. If Sarah can suggest a book but never see the results of a vote, the experience is incomplete.
- [ ] **Does each step in the journey make sense without the features you've cut?** Removing a detail shouldn't leave a gap in the flow. If it does, you may need that detail after all.

### Is it ready for feedback?

- [ ] **Could you show this to a real user this week?** If you're thinking "I just need to add one more thing first," that's a sign you're over-scoping.
- [ ] **Will you learn something from user reactions?** The MVP should test your core assumption about what the user needs. If the feedback would be the same regardless of how you build it, you're not testing anything yet.

### Quick comparison: MVP vs. too much

| MVP | Too much |
|---|---|
| Create club (enter club name) | Create club with description, rules, reading pace, and club type |
| Suggest books (add title and author) | Suggest books with cover image, recommendation note, genre tags, and reading level |
| Vote on books (cast vote, view counts) | Vote with comments, deadline reminders, notification emails, and weighted voting |

The left column solves the problem. The right column is what you build once you've confirmed the left column works.
