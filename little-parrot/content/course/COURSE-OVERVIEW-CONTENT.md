### Challenge bullet points

**Your First Prompt**
- Discover what vibe coding is and how Lovable turns your words into a working app

**Iterate and Improve**
- Apply small, focused follow-up prompts to improve your app step by step

**Save Data with Lovable Cloud**
- Connect your app to Lovable Cloud so data persists across sessions
- Approve database changes confidently by understanding what each migration does
- Set up Google login so your app knows who's using it

**Make Quick Changes with the Tools Lovable Provides**
- Preview your app on different screen sizes to see what users see
- Apply visual edits and design themes to refine your app's look and feel
- Use image attachments to show Lovable exactly what you want

**Experiment and Undo Mistakes Safely**
- Bookmark stable versions of your app before trying something risky
- Revert to a previous version when things break
- Experiment freely knowing you can always go back

**Plan, Test, and Connect Services**
- Use Plan Mode to think through complex features before building them
- Give Lovable lasting context about your project with Project Knowledge
- Connect external services using API keys and handle security warnings confidently

**Publish and Share Your App**
- Publish your app so anyone with the link can use it
- Set up a favicon, share image, and meta tags to make your app look professional
- Gather feedback from real users and use it to improve your product

### Short description
Stop panicking when your app breaks. Build a practical debugging toolkit so you can find what went wrong, fix it, and keep moving forward — even without a technical background.

### What you'll be able to do
- Identify and categorise different types of bugs in your app
- Prevent common bugs before they happen using simple habits
- Investigate issues step by step, the way professional developers do
- Use Lovable's built-in debugging tools to fix problems faster
- Escape frustrating fix-break-fix loops with specific strategies
- Handle security errors and warnings confidently

### Challenge bullet points

**The Anatomy of a Bug**
- Distinguish between visual bugs, logic bugs, and data bugs
- Develop a calm, investigative mindset when something breaks
- Categorise bugs by where they happen (frontend vs. backend)

**Preventing Bugs Before They Happen**
- Apply simple prompting habits that dramatically reduce frustrating bugs
- Build incrementally and test after each change
- Structure your prompts to prevent misunderstandings with Lovable

**How Developers Investigate Bugs**
- Apply the same investigation techniques professional developers use
- Reproduce a bug consistently before attempting a fix
- Isolate the root cause instead of guessing at random solutions

**Lovable's Debugging Toolkit**
- Use the browser console to spot errors you can't see on screen
- Navigate Lovable's error messages and understand what they mean
- Apply the right debugging tool for each type of problem

**Escaping Bug Loops**
- Recognise when you're stuck in a fix-break-fix cycle
- Apply specific strategies to break out of the loop and move forward
- Decide when to revert to a working version and try a different approach

**How to Guide Lovable to Fix Bugs**
- Write bug reports that help Lovable understand and fix the issue faster
- Provide the right context — screenshots, error messages, and steps to reproduce
- Communicate effectively so fixes stick on the first try

**Handling Security Errors and Warnings**
- Understand what Lovable's security warnings actually mean
- Evaluate which warnings need immediate action vs. which are safe to note
- Apply the right fix based on whether your app handles sensitive data

## 4. Build a Valuable Product: How to Break Down Your Idea for Vibe Coding

### Short description
Turn a messy app idea into a clear, buildable plan. You'll define who it's for, what problem it solves, and break it down into steps you can build with an AI app builder — starting with the smallest version that matters.

### What you'll be able to do
- Define the core problem your product solves using a clear problem statement
- Create a user persona that guides every product decision
- Visualise your user's journey from struggle to solution with a storyboard
- Break your idea into buildable steps using user story mapping
- Decide what to build first by shaping a focused MVP
- Turn your plan into a clear prompt for an AI app builder

### Challenge bullet points

**Craft a Problem Statement**
- Identify the core user need your product idea addresses
- Apply the problem statement template to articulate what you're solving
- Evaluate whether your problem is worth solving before investing time building

**Define Your Target User**
- Create a user persona that represents who you're building for
- Prioritise features by asking "would my target user need this?"
- Develop empathy for your user's frustrations and goals

**Storyboard: From Struggle to Solution**
- Visualise how your user goes from their current problem to your solution
- Map the key moments where your product makes a difference
- Identify gaps in your idea before you start building

**Break Down Your Idea Into Buildable Steps**
- Apply user story mapping to structure what users do in your app
- Organise features into logical groups of related actions
- Create a clear map that shows every interaction your app needs

**Decide What to Build First: Define Your MVP**
- Distill your idea down to the minimal set of features worth testing
- Prioritise ruthlessly — cut everything that isn't essential to the core experience
- Shape an MVP you can build and test within days, not months

**Create a Prompt to Build Your MVP with AI**
- Transform your user story map into structured building instructions
- Write a prompt that gives an AI app builder exactly what it needs
- Ship a first version of your product idea using what you've planned

**Get Feedback on What You've Built**
- Design a lightweight feedback plan to test your idea with real people
- Identify the right questions to ask your first testers
- Evaluate feedback to decide what to improve, add, or remove next

## 5. Basics of Software for Vibe Coding

### Short description
Learn the language of software so you can tell your AI app builder exactly what you want — and understand what it builds. No code, no jargon overwhelm, just the essential vocabulary explained through everyday analogies.

### What you'll be able to do
- Explain what frontend, backend, and databases do using friendly language
- Use correct design terminology to describe how you want your app to look
- Communicate more precisely with AI app builders like Lovable
- Understand what's happening behind the scenes when your app is built
- Identify which part of your app to reference when something needs changing

### Challenge bullet points

**Building Blocks of an Application**
- Distinguish between frontend, backend, and database — the three core parts of any app
- Understand how these parts work together (like a bakery's counter, kitchen, and storage)
- Apply this knowledge to give more targeted instructions to AI app builders

**Design Elements for Vibe Coders**
- Use design vocabulary (layout, spacing, typography, colour) to describe what you want
- Communicate visual changes precisely so you get it right on the first try
- Recognise common design patterns used in web applications

**Frontend for Vibe Coders**
- Identify key frontend concepts: components, pages, navigation, and responsive design
- Understand what your AI app builder does when it creates screens and interactions
- Give better feedback on layout and user interface issues

**Backend for Vibe Coders**
- Understand how features like login, email notifications, and payments work behind the scenes
- Recognise when a feature requires backend logic vs. frontend changes
- Prompt for powerful backend features with the right terminology

**Databases for Vibe Coders**
- Understand how your app stores, retrieves, and organises data
- Distinguish between different types of data and how they relate to each other
- Avoid common data-related mistakes that can cause bugs or data loss
