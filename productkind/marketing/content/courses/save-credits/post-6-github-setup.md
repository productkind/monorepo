# Post 6: Set up GitHub (your documents get a home too)

**Pillar:** the method, shown, with the beyond-code wedge.
**Research trace:** GitHub integration sits in our strongest demand data ("the
next steps after the first build"); the audience already uses AI for documents
(emails, meal plans), so the documents angle meets them where they are; the
AI-agent framing plants the on-ramp the course is built around; version
control is taught as the real term with the familiar Google Docs comparison,
straight from Challenge 1 (which is free, so this post points at the lead
magnet directly).

## Format

**Spec key for the designer:** In the slide table, any text introduced as
"Prompt:" (or "prompt Lovable:", "Say:", "Assembled:") or written with
markdown symbols (##, -, 1., backticks) is a prompt snippet: literal text
the learner types into a tool. Render it verbatim in monospace inside a
prompt-input mockup (a chat input field with a cursor and send arrow), so
it reads as typed text rather than decoration; markdown symbols are part of
the typed text, never styled labels. Everything else in "Text on slide" is
display copy in the brand style.

calm and concrete. A folder/repository visual motif carries through the
slides.

**Alt text (for the post):** An explainer carousel about GitHub: it stores
your work in repositories you can open from any computer, keeps a full
version history like Google Docs, works for documents as well as code, and,
together with version control on your computer, lets you review line-by-line
changes when an AI agent edits your files.

## Caption (exact)

We put a whole section about GitHub at the very start of our new course, because it may be the most broadly useful thing you set up all month. 💡

When you build with Lovable, GitHub is where your app's code lives. But the account does more than hold code:

☁️ your work is online, so you can open it from any computer
🕰️ every change is saved with a full history you can roll back (the real name for this is version control, and you already know the idea from Google Docs)
📄 documents live happily there too: your content calendar, your CV, a book chapter. Some people write whole books this way

It's also the perfect base for working with an AI agent like GitHub Copilot or Claude Code. Unlike a chat assistant you copy-paste from, an agent changes your files directly on your computer, and version control shows you what changed, line by line, so you stay in charge. When you're happy with it, you save it all to GitHub.

You set up your account and connect your app in the first challenge of Save Lovable Credits: Edit Your App Like a Developer, and that challenge is free. You'll find the course on LittleParrot.app.
