# Post 3: Swap any icon for free (micro-tutorial)

**Pillar:** the method, shown.
**Research trace:** real steps beat results-without-method (the audience's
stated frustration); a small, definite win ("icons are just code you can
rename") is exactly the concrete-outcome framing that closes the confidence
gap; saveable micro-tutorial format.

## Format

**Spec key for the designer:** In the slide table, any text introduced as
"Prompt:" (or "prompt Lovable:", "Say:", "Assembled:") or written with
markdown symbols (##, -, 1., backticks) is a prompt snippet: literal text
the learner types into a tool. Render it verbatim in monospace inside a
prompt-input mockup (a chat input field with a cursor and send arrow), so
it reads as typed text rather than decoration; markdown symbols are part of
the typed text, never styled labels. Everything else in "Text on slide" is
display copy in the brand style.

**Carousel, 6 slides, 1080 x 1350 (4:5 portrait).** Step-by-step tutorial
style: each slide is one numbered step with a cropped screenshot (browser
Inspect panel, VS Code search, lucide.dev) and one line of instruction. Heart
icon changing to a star as the running example.

| Slide | Visual | Text on slide |
| --- | --- | --- |
| 1 (cover) | A heart icon with an arrow to a star icon, big type | "That icon in your Lovable app? Swap it yourself, for free. Icons are code, with names you can change." |

**Alt text (for the post):** A step-by-step tutorial carousel showing how to
swap an icon in a Lovable app for free: find its name with the browser's
Inspect tool, search it in VS Code, pick a new icon on lucide.dev, and rename
it in the code.
